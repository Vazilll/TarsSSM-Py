# Инструкция по развертыванию TARS (GitHub + 2-й ПК)

## Шаг 1: Подготовка репозитория (на 1-м ПК)
1. Инициализируйте Git в папке `TarsSSM(py)`:
   ```bash
   git init
   git add .
   git commit -m "Initial commit: TARS Ultimate Stack Python"
   ```
2. Создайте пустой репозиторий на GitHub и свяжите его:
   ```bash
   git remote add origin https://github.com/ВАШ_ЛОГИН/TarsSSM-Py.git
   git push -u origin main
   ```

## Шаг 2: Установка на 2-м ПК
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/ВАШ_ЛОГИН/TarsSSM-Py.git
   cd TarsSSM-Py
   ```
2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   ```
3. Установите все библиотеки одной командой:
   ```bash
   pip install -r requirements.txt
   ```

## Шаг 3: Перенос моделей
Тяжелые файлы моделей (`.gguf`, `.pt`, `.onnx`) **не попадут** в Git автоматически из-за их размера (и лимитов GitHub).
1. Скопируйте папку `models/` с первого ПК.
2. Перенесите её на второй ПК (через сетевую папку, флешку или внешний диск).
3. Поместите папку в корень проекта `TarsSSM(py)`.

## Шаг 4: Запуск квантования (на 2-м ПК)
Теперь используйте мощности второго компьютера, чтобы перевести модели в формат BitNet (1.58-бит):
```bash
python scripts/quantize_models.py
```
*Скрипт автоматически определит наличие видеокарты (CUDA) и задействует её для ускорения процесса.*

---
**Примечание**: Если ваш второй ПК имеет доступ к тем же локальным путям (C:\Users\Public\Tarsfull), вы можете просто запустить `python setup_tars.py` для автоматического подтягивания моделей.
